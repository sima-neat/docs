Local Setup

In the standalone setup configuration, the Modalix DevKit 3.0 is used like a Linux PC. No network connection is required. Click on each item for more information.


Connect the Power supply to the 12V Power input and power up the DevKit.

Login & Run the Demo

  1. Use the default username sima and password edgeai to log in.
  2. Run the Demo
    1. This demo requires a camera and a speaker.
    2. Click the “Demo” icon on the desktop to launch the demo.
    3. Ensure that you see a live camera image.
    4. Ask a question in the chat, for example: “What do you see in the image?”

Remote Setup

Configure your DevKit for remote access using an external PC or Laptop. An Ethernet cable (not included in the Devkit) is required. The Devkit will be accessed through a web browser to operate a Demo.

How to run a demo: Once connected, open http://modalix.local:8800 or use the DevKit’s IP address in Chrome to access the remote web UI.

Prepare for Development

Once your DevKit is connected and powered up, you can begin developing with the SiMa Modalix DevKit 3.0.

1️⃣ Sign Up on the SiMa Developer Portal

2️⃣ Install the SiMa CLI

Download and install sima-cli on your host machine to access the SDK and developer tools. Follow the Installation Guide.

Access our technical documentation for hardware, software, and code examples at developer.sima.ai.

Troubleshooting

Common issues and solutions to help you get your SiMa DevKit running smoothly.

Ensure the 12V power adapter is securely connected and the outlet is active. The red power LED on the board should illuminate when properly powered.

Make sure your laptop and the DevKit are connected to the same network. Try opening http://modalix.local:8800 in Chrome.

Note:
The modalix.local address relies on mDNS, which may not work on all operating systems (especially Windows).

If modalix.local is not reachable:

  1. After installing sima-cli, run sima-cli device discover on your host machine (while connected to the DevKit). This command lists all detected DevKits and their IP addresses.
  2. If no device appears, connect to the DevKit’s serial console from your host machine by running: sima-cli serial
  3. Log in with username sima and password edgeai, then type ifconfig to display the assigned IP address.
  4. The same IP address can also be found under the Settings tab in the Local Setup interface once the DevKit is running.

If sima-cli cannot discover your DevKit, it's possible that your DevKit runs an older version of the firmware. Check the next topic to learn how to update the DevKit.

Depending on the manufacturing date, your DevKit may be running an older firmware version that does not match the demo procedure described above. If your DevKit is running version 1.7, it must be converted to the 2.0 eLxr release before you can follow the Quick Start guide and try out the demos.

To check your current firmware version and determine whether conversion is required, please follow the instructions in the conversion guide below.

DevKit 1.7 → 2.0 eLxr Conversion Guide

The out-of-the-box demo experience on the Modalix DevKit can be updated directly on the device without reflashing the system image. This is supported on DevKits running the eLxr release.

To update the demo suite, log in to your eLxr DevKit and run the following command: 

cd /tmp && curl -O https://tools.modalix.info/setup-oob.sh && bash setup-oob.sh

This script downloads and installs the latest version of the demo suite, ensuring that your device is up to date with the current out-of-the-box experience.

Verify that your monitor is powered on and set to the correct HDMI input. Reconnect the HDMI cable and reboot the DevKit if necessary.
Use the default credentials — username sima and password edgeai. If these were changed, contact your administrator or SiMa support.

Install the Card

Install the PCIe card and connect bring-up interfaces before software setup.

Step 1: Power down the host and open the chassis.
Step 2: If the motherboard has both black and blue PCIe slots, install the SiMa card in a black slot.
Step 3: Secure the bracket and reassemble the chassis.
Step 4: It is recommended to connect the serial connector from card to host for debug access.
Step 5: The PCIe card also includes a physical Ethernet port for direct network access when needed.

Host Setup

Install the host driver package based on your host OS.

Prerequisite: Install sima-cli first on your host machine. Follow the setup guide: sima-cli installation.
Linux host:
sima-cli install drivers/linux
Windows host:
sima-cli install drivers/windows
After install: Power cycle the host machine before validation.

Verify and Test

Use device discovery to confirm PCIe enumeration and connectivity.

Step 1: On the host side, run:
sima-cli device discover
Note: PCIe virtual ethernet throughput testing is currently available on Linux hosts only.
